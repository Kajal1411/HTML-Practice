Que.1 What is frontend?

Frontend:

What it is: The part of the website that users see and interact with.
Responsibilities: Handles the design, layout, and interactivity.
Technologies:HTML, CSS, JavaScript.
Runs on: Browser (client-side).

Que.2 What is backend?

Backend:

What it is:The server-side of the website that processes data and logic.
Responsibilities: Manages data, databases, and server logic.
Technologies: Node.js, Python, Java, PHP.
Runs on:Server (server-side).

Que.3 What is static webpage?

A static webpage is a type of webpage that displays the same content to every visitor. The content of a static webpage is fixed and doesn't change unless manually updated by the website's developer. These pages are usually written in HTML (HyperText Markup Language), and they do not have any interactive or dynamic features, such as forms or databases.

Que.4 What is dynamic webpage?

A dynamic webpage is a webpage that can display different content based on user interactions, preferences, or real-time data. Unlike static webpages, dynamic webpages can change their content and appearance without requiring a full page reload. These pages are often generated using server-side programming languages (such as PHP, Python, or Node.js) and may involve client-side scripting (like JavaScript) to make the page interactive.

Que.5 what is client-server mechanism?

The client-server mechanism for frontend involves the communication between the user interface (frontend) and the server (backend). When a user interacts with the frontend, like clicking a button or submitting a form, the frontend sends a request to the server. The server processes the request, often querying a database, and sends back a response (like data in JSON format). The frontend then updates the user interface based on this response, providing a dynamic experience without reloading the entire page. This mechanism helps keep the user experience smooth and interactive by separating the user interface from the data processing.

Que.6 What is HTML?

HTML (HyperText Markup Language) is the standard language used to create and design webpages. It provides the structure of a webpage by using a system of elements (tags) to define content types, such as headings, paragraphs, links, images, tables, forms, and more.

Que.7 What is mean by HyperText?

Hypertext refers to text that contains links (known as hyperlinks) to other documents, webpages, or resources on the internet. These links allow users to navigate easily between related pieces of information by clicking on them.

Que.8 What is mean by Markup language?

A Markup Language is a system for annotating or "marking up" text in a document to define its structure and presentation. The annotations are written using tags or codes that are interpreted by a computer to format, structure, or display the content.

Que.9 Who developed HTML?

HTML (HyperText Markup Language) was developed by Tim Berners-Lee, a British computer scientist, in 1991. He created it while working at CERN (European Organization for Nuclear Research) as a way to share and link documents over the internet.

Que.10 Features of HTML.

1.Markup Language: HTML is a markup language that structures and organizes content on web pages using various tags.

2.Tags and Elements: HTML uses tags to define content types like headings, paragraphs, links, and images. Each element is usually made up of an opening and closing tag.

3.Structure and Organization: HTML provides the essential structure of a web page, typically consisting of a head, body, and footer, helping organize content for browsers to render.

4.Hyperlinks: HTML allows the creation of hyperlinks to navigate between pages and websites, making it interactive.

5.Embedding Media: HTML supports embedding media files like images, audio, video, and interactive content to enhance user experience.

6.Forms: HTML allows the creation of interactive forms to collect data from users, such as input fields, text areas, and buttons.

7.Tables: HTML can structure tabular data using rows and columns, making it easy to display information in an organized manner.

8.Semantic HTML: It includes semantic tags that provide meaning to content, such as for headers, articles, sections, and footers. This improves both accessibility and SEO.

9.Attributes: Tags can have attributes that provide additional details or define behaviors, such as class, id, or source URLs.

10.Document Structure: HTML documents start with a document type declaration and contain a head section for meta information and a body section for content display.

11.Browser Compatibility: HTML is universally supported by all modern web browsers, ensuring consistent display across platforms.

12.Responsive Design: HTML can be paired with CSS and media queries to create responsive layouts, making websites work well on devices of various screen sizes.
